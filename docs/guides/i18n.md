# 国际化 (i18n) 使用指南

本项目支持中英文国际化，基于 Chrome Extension i18n API 实现。

## 文件结构

```

├── public/
│   └── _locales/
│       ├── en/
│       │   └── messages.json     # 英文翻译
│       └── zh_CN/
│           └── messages.json     # 中文翻译
├── src/
│   ├── utils/
│   │   └── i18n.ts              # i18n 工具函数
│   └── hooks/
│       └── useI18n.tsx          # React Hook
└── manifest.json                 # 使用 __MSG__ 格式
```

## 使用方法

### 1. 在 manifest.json 中使用

```json
{
  "name": "__MSG_extension_name__",
  "description": "__MSG_extension_description__",
  "default_locale": "en"
}
```

### 2. 在 React 组件中使用

```tsx
import { useI18n } from '../hooks/useI18n';

function MyComponent() {
  const { t } = useI18n();

  return (
    <div>
      <h1>{t('popup_title')}</h1>
      <button>{t('settings_save')}</button>
    </div>
  );
}
```

### 3. 在 TypeScript/JavaScript 代码中使用

```typescript
import { getMessage } from '../utils/i18n';

const message = getMessage('error_no_api_key');
console.log(message);
```

## 添加新的翻译

### 步骤 1：添加到 messages.json

在 `public/_locales/en/messages.json` 和 `public/_locales/zh_CN/messages.json` 中添加：

```json
{
  "new_key": {
    "message": "Your text here",
    "description": "Description of this message"
  }
}
```

### 步骤 2：在代码中使用

```tsx
const { t } = useI18n();
return <div>{t('new_key')}</div>;
```

## 语言切换

用户可以在设置页面选择语言：

```tsx
import { useI18n } from '../hooks/useI18n';

function SettingsPage() {
  const { changeLanguage, currentLang } = useI18n();

  return (
    <select
      value={currentLang}
      onChange={(e) => changeLanguage(e.target.value as any)}
    >
      <option value="auto">Auto (Browser Language)</option>
      <option value="en">English</option>
      <option value="zh_CN">中文</option>
    </select>
  );
}
```

## 命名规范

使用下划线命名法，按模块分类：

- `extension_*` - 扩展相关
- `popup_*` - 弹窗页面
- `settings_*` - 设置页面
- `history_*` - 历史记录
- `chat_*` - 聊天相关
- `action_*` - 操作按钮
- `model_*` - 模型相关
- `error_*` - 错误消息
- `confirm_*` - 确认对话框
- `tooltip_*` - 提示文本

示例：
- `popup_title` - 弹窗标题
- `settings_save` - 保存按钮
- `error_no_api_key` - 无 API Key 错误

## 测试

### 1. 测试不同语言

在浏览器中：
1. 打开 `chrome://settings/languages`
2. 添加或调整语言顺序
3. 刷新插件页面

### 2. 开发环境

开发时如果 `chrome.i18n` 不可用，会自动降级返回 key 名称。

## 最佳实践

1. **避免硬编码文本**
   ```tsx
   // ❌ 不好的做法
   <button>保存</button>

   // ✅ 好的做法
   <button>{t('settings_save')}</button>
   ```

2. **使用描述性 key 名称**
   ```json
   // ❌ 不好的做法
   "btn1": { "message": "Save" }

   // ✅ 好的做法
   "settings_save": {
     "message": "Save",
     "description": "Save button in settings page"
   }
   ```

3. **保持翻译一致性**
   - 同一概念使用相同术语
   - 按钮文案保持简洁

4. **添加占位符支持**（如需要）
   ```json
   "greeting": {
     "message": "Hello, $USER$!",
     "placeholders": {
       "user": {
         "content": "$1",
         "example": "John"
       }
     }
   }
   ```

   使用：
   ```tsx
   getMessage('greeting', ['John'])
   ```

## 常见问题

### Q: 为什么我的翻译没有生效？

A: 检查以下几点：
1. `messages.json` 文件格式是否正确
2. key 名称是否正确
3. `default_locale` 是否在 manifest.json 中设置
4. 刷新插件

### Q: 如何调试国际化？

A:
```typescript
console.log('Current language:', currentLang);
console.log('Message:', t('my_key'));
```

### Q: 支持更多语言？

A: 添加新的 `_locales/语言代码/messages.json` 文件即可。
