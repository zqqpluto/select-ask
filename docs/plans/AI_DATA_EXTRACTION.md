# AI 取数功能 — 功能设计文档

## 概述

在 Select Ask 浏览器扩展中新增一个独立的 **AI 取数** 页面，让用户能够从网页或粘贴文本中提取结构化数据，自动生成取数脚本、执行分析，并基于结果推荐可视化图表。

**核心价值**：低门槛地将网页中的散乱信息转化为可分析的结构化数据，并快速生成图表。

---

## 页面架构

```
┌──────────────────────────────────────────────────────────┐
│  Select Ask — AI 取数                                      │
├──────────────────────────────────────────────────────────┤
│  [步骤1] 输入数据 → [步骤2] 取数规则 → [步骤3] 结果 & 图表 │
└──────────────────────────────────────────────────────────┘
```

页面采用 **分步引导式** 设计，三个步骤通过顶部 Tab/进度条切换：

1. **数据输入** — 选择数据来源
2. **取数规则** — 描述取数需求，AI 生成脚本并预览
3. **结果分析** — 结构化数据表格 + 推荐图表

**粘贴文本数据**：
- 大面积文本输入区域（最小 200px 高，可拉伸）
- 自动识别格式（CSV、JSON、TSV、Markdown 表格）并显示提示
- 显示预估行数，帮助用户确认粘贴成功

### 交互细节

**输入取数需求**：
- 自然语言输入，支持中文
- 快捷模板按钮：点击后在输入框填入预设的自然语言模板，用户可修改

**生成取数脚本**：
- AI 基于步骤 1 的数据 + 用户的自然语言需求，生成取数脚本
- 先展示 **示例数据（前 5 行）** 让用户确认结果符合预期
- 同时展示 **取数脚本**（默认折叠，可展开查看/编辑）
- 脚本以安全的 JS 函数形式执行（沙箱环境，仅对内存数据操作，无 DOM/File/Network 访问权限）

**可编辑脚本**：
- 用户可以直接修改 AI 生成的脚本
- 修改后点击"执行"即时预览结果

## 入口设计

### 方式 1：悬浮图标菜单

在现有的悬浮图标菜单中新增 **"AI 取数"** 选项，点击后在新 tab 中打开取数页面。

### 方式 2：右键菜单

注册浏览器右键菜单（当页面检测到表格时显示）：
`右键 → Select Ask → 提取页面数据`

### 推荐方案

**方式 1 + 方式 2** 组合：悬浮图标覆盖全场景，右键菜单在检测到表格时提供快捷入口。

## 技术实现要点

### 1. 页面数据检测

```typescript
interface DetectedData {
  type: 'table' | 'list' | 'card';
  source: string;
  columns: string[];
  rows: Record<string, string>[];
  rowCount: number;
}
```

- 遍历页面 `<table>` 元素，解析 `<thead>`/`<tbody>`
- 对于列表/卡片，使用启发式方法检测重复 DOM 结构

### 2. 脚本执行沙箱

- 脚本仅对内存中的数据进行 filter/map/reduce 等操作
- 注入的上下文仅有 source 数据变量
- 无 DOM/File/Network 访问权限

### 3. AI 提示词设计

**取数脚本生成**：AI 接收示例数据（JSON）+ 用户需求（自然语言），输出 JS 取数脚本

**推荐分析生成**：AI 接收列 Schema 信息，输出 3-5 个推荐分析（标题、描述、图表类型）

**图表生成**：AI 接收列名 + 用户需求，输出图表配置

### 4. 文件格式导出

- CSV：数组转逗号分隔字符串
- JSON：格式化输出
- 使用 Blob + URL.createObjectURL 触发下载

## 优先级建议

**MVP（第一阶段）**：
- 粘贴文本数据输入（CSV/JSON 自动解析）
- 自然语言描述 → AI 生成脚本 → 执行 → 展示结果表格
- 3 个推荐分析 + 1 个自定义图表生成
- 导出 CSV/JSON

**第二阶段**：
- 页面表格自动检测与提取
- 右键菜单快捷入口
- 可编辑脚本
- 更多图表类型

**第三阶段**：
- 数据清洗与多源合并
- 定时取数
