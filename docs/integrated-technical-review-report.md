## 一、综合风险评估

### 1.1 严重安全风险 🔴

#### 问题: Chrome ID可被伪造

**当前实现**:
```typescript
// browser-extension/src/models/Device.ts
device_id: crypto.randomUUID() // 存储在chrome.storage.sync
```

**攻击场景**:
```javascript
// 攻击者可以轻松绕过限流
for (let i = 0; i < 1000; i++) {
  // 清除存储,生成新ID
  chrome.storage.sync.clear();
  const newId = crypto.randomUUID();
  chrome.storage.sync.set({ device_id: newId });
  
  // 每个新ID获得30次免费额度
  // 1000个ID = 30,000次免费调用
}
```

**影响量化**:
- 单个恶意用户每天可消耗: 30,000次API调用
- 假设DeepSeek成本 ¥0.01/次
- 单用户单日成本: ¥300
- 月成本: ¥9,000
- **100个恶意用户 = ¥900,000/月** ⚠️

**风险等级**: Critical - 必须立即解决

---

### 1.2 解决方案: 设备指纹技术

#### 方案对比

| 方案 | 安全性 | 可靠性 | 实施难度 | 用户体验 | 推荐度 |
|------|--------|--------|----------|----------|--------|
| 当前UUID | ❌ 低 | ❌ 低 | ✅ 低 | ✅ 好 | ❌ |
| Chrome Instance ID | ⚠️ 中 | ✅ 高 | ⚠️ 中 | ✅ 好 | ⚠️ |
| 设备指纹 | ✅ 高 | ✅ 高 | ⚠️ 中 | ✅ 好 | ✅ |
| 用户账号 | ✅ 最高 | ✅ 最高 | ❌ 高 | ❌ 差 | ⚠️ |

#### 推荐方案: 多维度设备指纹

**技术架构**:
```typescript
interface DeviceFingerprint {
  canvas: string;      // Canvas指纹
  webgl: string;       // WebGL渲染器信息
  audio: string;       // AudioContext指纹
  screen: string;      // 屏幕分辨率+色彩深度
  timezone: string;    // 时区
  language: string;    // 语言
  platform: string;    // 操作系统
  plugins: string[];   // 浏览器插件列表
  fonts: string[];     // 字体列表
  hash: string;        // 综合hash
}
```

**安全保障**:
- ✅ 伪造难度高:需要同时伪造多个硬件特征
- ✅ 准确率高:95%+设备识别准确率
- ✅ 隐私友好:不收集个人身份信息
- ✅ 稳定性好:清除浏览器数据不会改变
